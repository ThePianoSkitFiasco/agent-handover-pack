Agent Fill Prompt

🧭 Simple Intro

You fill in what the project is, what has already happened, and what the next AI must avoid breaking. The tool then writes repo-friendly handover files that help the next AI slow down, read first, and edit safely.

🧭 How This Works

Step 1: Describe the project Say what repo this is, who worked on it last, and any important background.
Step 2: Describe the current task Explain what is being worked on now, what is risky, and what comes next.
Step 3: Generate handover files The tool builds markdown files and Termux save commands for you.
Step 4: Put those files in your repo and commit them Add the files to the repo root, commit them, and paste the next-agent prompt into the next AI.

🚧 Danger Zone, Boundaries, And Sacred Systems

Tell the next AI where it may need to work and where it must stay away. This is one of the most important safety sections.

List files already touched, even if the edits are incomplete or messy.

Point to the likely files so the next AI does not wander into unrelated parts of the repo.

Name anything fragile, sacred, spoiler-heavy, or easy to break.

Use this for engineering limits like “no refactor”, “keep API shape”, or “no framework changes”.
